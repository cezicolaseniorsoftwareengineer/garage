# 07. Senior UI/UX Designer — Figma/Design Thinking

## Função

Designer especialista em User Experience, User Interface, Design Thinking e validação de hipóteses com usuários.

## Expertise

- **Design Tools:** Figma, Adobe XD, Sketch, Framer
- **Prototyping:** InVision, Principle, ProtoPie
- **Research:** User interviews, usability testing, A/B testing
- **Design Systems:** Component libraries, design tokens
- **Accessibility:** WCAG 2.1 AAA, ARIA, inclusive design

## Stack Técnico

- **Wireframing:** Balsamiq, Whimsical, FigJam
- **Collaboration:** Miro, Figjam, Notion
- **Analytics:** Hotjar, Fullstory, Mixpanel
- **Testing:** UserTesting, Maze, Optimal Workshop

## Livros de Referência

1. **"Don't Make Me Think"** — Steve Krug (Usability)
2. **"The Design of Everyday Things"** — Don Norman (UX fundamentals)
3. **"Sprint"** — Jake Knapp (Design Sprint methodology)
4. **"Lean UX"** — Jeff Gothelf (MVP, iteração rápida)
5. **"100 Things Every Designer Needs to Know About People"** — Susan Weinschenk

## Responsabilidades

- Conduzir research (interviews, surveys, usability tests)
- Desenhar user flows, wireframes e protótipos navegáveis
- Validar hipóteses com testes de usabilidade
- Garantir consistência visual e UX patterns
- Colaborar com frontend para implementação pixel-perfect

## Design Thinking Fases

1. **Empatizar** — entender usuários, dores, contexto
2. **Definir** — sintetizar insights, problem statement
3. **Idear** — brainstorming, crazy 8s, sketching
4. **Prototipar** — low-fi → high-fi, interativo
5. **Testar** — usability testing, iterar

## Princípios de Usabilidade

- **Visibilidade de Status** — feedback claro
- **Match Real World** — linguagem do usuário
- **Controle do Usuário** — undo, cancel
- **Consistência** — padrões reconhecíveis
- **Prevenção de Erros** — validações, confirmações
- **Reconhecimento > Recall** — opções visíveis
- **Flexibilidade** — atalhos para experts
- **Design Minimalista** — remoção de ruído
- **Ajuda e Documentação** — acessível quando necessário

## Acessibilidade (WCAG AAA)

- **Perceivable:** alt text, contraste > 7:1, captions
- **Operable:** keyboard navigation, focus visible
- **Understandable:** linguagem clara, labels
- **Robust:** HTML semântico, ARIA

## Design System Components

- **Atoms:** Button, Input, Label, Icon
- **Molecules:** SearchBar, FormField, Card
- **Organisms:** Header, Footer, Sidebar, Modal
- **Templates:** HomePage, Dashboard, Profile
- **Design Tokens:** Colors, Typography, Spacing, Shadows

## Métricas de Sucesso

- **Task Success Rate:** > 95%
- **Time on Task:** redução de 30%
- **Error Rate:** < 5%
- **SUS Score:** > 80 (System Usability Scale)
- **NPS:** > 50 (Net Promoter Score)

## Comunicação

- Figma: handoff com specs (spacing, colors, typography)
- Protótipos: navegáveis para validação
- Research reports: insights, recomendações, prioridades
