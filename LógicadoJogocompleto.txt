Nome do jogo

GARAGE – Toda Big Tech tem um início

O nome “GARAGE” representa o ponto zero da engenharia:
o primeiro código, o primeiro erro, o primeiro sistema imperfeito —
antes da escala, antes do sucesso, antes da fama.

Objetivo central (imutável)

Permitir que qualquer pessoa evolua de um simples Hello World até o nível de Senior Software Engineer, dominando fundamentos reais de engenharia de software, e não dependência de frameworks, modismos ou atalhos superficiais.

O jogo não ensina “como usar tecnologia”.
Ele ensina como pensar como engenheiro.

Tipo de jogo

Single Player

Progressão por carreira

Baseado em desafios técnicos reais

Totalmente orientado a backend, lógica, dados e arquitetura

Não há foco em UI chamativa.
Não há mecânicas de distração.
O desafio é intelectual, técnico e conceitual.

Plataforma

Web

HTML / CSS / JavaScript apenas como camada de visualização

Backend

Python + FastAPI (núcleo do jogo)

Banco de Dados

Supabase (PostgreSQL + Auth + Storage)

PRINCÍPIO FUNDAMENTAL (REGRA ABSOLUTA)

Este não é um jogo de frontend.

O frontend não decide nada.
O frontend não valida nada.
O frontend não possui regras de negócio.

Tudo que importa acontece no backend.

O frontend apenas renderiza o estado do jogo, exatamente como um cliente HTTP em um sistema real.

Esse princípio educa o jogador desde o primeiro minuto sobre separação de responsabilidades e arquitetura correta.

3.2 Sistema de Erros (Regra Dura)

Cada desafio permite 3 erros

3 erros → Game Over

Consequências do Game Over

Retorno ao início do estágio atual

Histórico mantido (aprendizado nunca é apagado)

Não existe “pagar para continuar”

O jogo ensina responsabilidade técnica e consequências reais.
